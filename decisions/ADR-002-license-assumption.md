# ADR-002: Initial License Assumption

Date: 2026-05-30

Updated: 2026-05-31

## Status

Accepted for initial public publication.

## Context

The user requested a professional open-source repository with a license file but did not specify a license.

## Decision

Use MIT as the initial project license.

## Consequences

- The repository has a concrete open-source license file.
- The public publication gate may proceed after validation and repository metadata updates.
- Future relicensing requires a new explicit decision.

## Alternatives Considered

- Apache-2.0: stronger patent language but more text and policy implications.
- Dual MIT/Apache-2.0: common in Rust but should be explicitly approved.
- License TBD: safer legally but less complete for an open-source scaffold.
